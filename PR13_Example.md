# Практическая работа

# Автоматизация десктоп-приложения с использованием Machine Vision в Puzzle RPA

---

# 📖 Ситуация

В компании **TechFlow** используется настольное приложение для регистрации и обработки сервисных заявок сотрудников.

Приложение позволяет сотрудникам:

* создавать новые заявки;
* отслеживать их статус;
* управлять обработкой заявок через таблицу.

В корпоративной инфраструктуре приложение может запускаться:

* через удалённые рабочие столы;
* внутри виртуальных рабочих сред;
* через терминальные серверы.

В таких условиях традиционная автоматизация интерфейса на основе **UI-селекторов** может быть недоступна или нестабильна.

Поэтому используется подход **Machine Vision**, при котором робот:

* анализирует изображение интерфейса;
* распознаёт текст и элементы управления;
* выполняет действия по координатам найденных объектов.

В рамках данной практической работы необходимо разработать **RPA-процесс в Puzzle RPA**, взаимодействующий с интерфейсом приложения **исключительно через визуальное распознавание элементов**.

---

# 🖥 Используемое приложение

Автоматизация выполняется для настольного приложения регистрации заявок.

Приложение состоит из **формы создания заявки**, **таблицы заявок** и **кнопок управления записями**.

---

# Форма регистрации заявки

Форма предназначена для **создания новой сервисной заявки** сотрудником.

Пользователь заполняет необходимые данные и добавляет заявку в систему.

---

## Поле Request Title

Назначение элемента:

поле используется для ввода **краткого названия или темы заявки**.

Примеры значений:

* Access to CRM
* Setup workstation
* VPN connection

При автоматизации робот должен вводить в это поле значение из Excel-файла.

---

## Поле Employee

Назначение элемента:

поле содержит имя сотрудника, который создаёт заявку.

Это поле используется системой для:

* идентификации автора заявки;
* последующего взаимодействия с сотрудником.

Примеры значений:

* Ivan Petrov
* Anna Smirnova

Робот должен автоматически вводить данные из входного файла.

---

## Список Priority

Назначение элемента:

данный элемент позволяет указать **приоритет обработки заявки**.

Это значение используется для определения срочности выполнения задачи.

Возможные значения:

* **Low** — низкий приоритет
* **Medium** — средний приоритет
* **High** — высокий приоритет

Заявки с высоким приоритетом должны обрабатываться в первую очередь.

В процессе автоматизации робот выбирает значение из списка.

---

# Кнопки формы

---

## Кнопка Add Request

Назначение элемента:

добавляет новую заявку в таблицу системы.

После нажатия кнопки:

* данные из формы сохраняются;
* в таблице заявок появляется новая строка;
* статус заявки устанавливается автоматически:

```
New
```

Робот должен нажимать эту кнопку после заполнения формы.

---

## Кнопка Clear Form

Назначение элемента:

очищает все поля формы ввода.

Используется пользователем для быстрого сброса введённых данных.

В рамках данной практической работы эта кнопка **не используется**, однако она может присутствовать в интерфейсе.

---

# Таблица заявок

Таблица расположена в нижней части окна приложения.

Она предназначена для **отображения всех зарегистрированных заявок**.

Каждая строка таблицы соответствует одной заявке.

Структура таблицы:

| ID | Title | Employee | Priority | Status |

---

## Назначение столбцов

**ID**

уникальный номер заявки.

---

**Title**

название заявки, введённое в поле Request Title.

---

**Employee**

имя сотрудника, создавшего заявку.

---

**Priority**

приоритет заявки.

---

**Status**

текущее состояние заявки.

Возможные значения:

```
New
Done
```

---

# Кнопки управления заявками

Эти кнопки используются для **обработки записей таблицы**.

Перед выполнением действий необходимо выбрать строку таблицы.

---

## Кнопка Mark as Done

Назначение элемента:

изменяет статус выбранной заявки.

После нажатия кнопки значение столбца **Status** изменяется на:

```
Done
```

Данная операция означает, что заявка обработана.

В процессе автоматизации робот должен:

* находить заявки с приоритетом **High**;
* выбирать соответствующую строку;
* нажимать эту кнопку.

---

## Кнопка Delete Selected

Назначение элемента:

удаляет выбранную заявку из таблицы.

После нажатия кнопки появляется окно подтверждения удаления.

Эта операция используется для удаления ошибочно созданных или устаревших заявок.

---

# Окно подтверждения удаления

Перед удалением записи приложение отображает диалоговое окно.

Текст сообщения:

```
Are you sure you want to delete this request?
```

Назначение окна:

предотвращение случайного удаления данных.

Для завершения операции пользователь должен подтвердить действие.

В рамках автоматизации робот должен:

* распознать появление этого окна;
* найти кнопку подтверждения;
* выполнить клик.

# УРОВЕНЬ 1 — Базовая визуальная автоматизация

Цель уровня — научиться **распознавать элементы интерфейса и взаимодействовать с ними через Machine Vision**.

---

# Шаг 1. Запуск приложения

## Зачем выполняется шаг

Перед выполнением автоматизации необходимо запустить приложение и убедиться, что интерфейс полностью загрузился.

## Что необходимо сделать

1. Запустить приложение через блок:

**System → Run Application**

2. Выполнить ожидание загрузки интерфейса.

Для этого использовать блок:

**Machine Vision → Find Text On Screen**

Параметры поиска:

```
Request Title
```

Результат шага:

робот получает координаты найденного текста и подтверждает, что интерфейс загружен.

---

# Шаг 2. Чтение входных данных

## Зачем выполняется шаг

Для автоматического создания заявок необходимо получить данные из Excel-файла.

## Что необходимо сделать

1. Использовать блок:

**Excel → Read Range**

2. Считать таблицу из файла:

```
requests.xlsx
```

3. Сформировать коллекцию записей для обработки.

Количество записей:

```
10
```

---

# Шаг 3. Поиск поля Request Title

## Зачем выполняется шаг

Робот должен определить расположение поля ввода, в которое будет вводиться название заявки.

## Что необходимо сделать

1. Использовать блок:

**Machine Vision → Find Text On Screen**

Параметр:

```
Request Title
```

2. После нахождения текста определить координаты области справа от найденной надписи.

3. Выполнить клик в этой области через блок:

**Mouse → Click**

4. Ввести текст через блок:

**Keyboard → Type Text**

Значение берётся из текущей строки Excel.

---

# Шаг 4. Заполнение поля Employee

## Зачем выполняется шаг

Необходимо указать сотрудника, создавшего заявку.

## Что необходимо сделать

1. Использовать блок:

**Machine Vision → Find Text On Screen**

Текст поиска:

```
Employee
```

2. Определить область поля ввода рядом с найденной меткой.

3. Выполнить клик:

**Mouse → Click**

4. Ввести текст:

**Keyboard → Type Text**

---

# Шаг 5. Выбор значения Priority

## Зачем выполняется шаг

Необходимо задать приоритет заявки.

## Что необходимо сделать

1. Найти элемент интерфейса через блок:

**Machine Vision → Find Text On Screen**

Текст поиска:

```
Priority
```

2. Выполнить клик по области выпадающего списка:

**Mouse → Click**

3. Использовать блок:

**Machine Vision → Find Text On Screen**

для поиска значения:

```
Low / Medium / High
```

4. После обнаружения нужного значения выполнить:

**Mouse → Click**

---
